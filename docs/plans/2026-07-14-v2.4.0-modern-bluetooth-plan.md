# v2.4.0 Modern Bluetooth Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship pymadoka 0.3.0 (HA-native BLE, lean packaging, tests) and daikin_madoka 2.4.0 (coordinator, discovery, dual setpoint, diagnostics, fr translation) as one coordinated release.

**Architecture:** pymadoka keeps owning the BLE protocol (transport chunking, feature commands) and gains first-class HA-stack connection support; the integration moves all polling behind one `DataUpdateCoordinator` per device and consumes only parsed feature statuses. Discovery, options, diagnostics are pure integration concerns.

**Tech Stack:** Python 3.12, bleak + bleak-retry-connector, HA 2025.1+ config entries / bluetooth helpers, pytest (pymadoka only), ruff.
